## Androidウィジェット

- Androidホーム画面ウィジェットで対象グループの直近の旅程を表示する
  - Androidウィジェットの実装方針をJetpack Glance + home_widgetにする
  - ウィジェットUIはAndroidネイティブのJetpack Glanceで実装する
  - 対象グループID、表示中旅行、過去1件、未来3件の旅程キャッシュはhome_widget経由で端末ローカルに保存する
  - DB取得とキャッシュ更新はFlutter側のユースケースで行う
  - 旅行名、旅行期間、旅程項目名、開始日時、終了日時、メモを表示対象にする
  - 旅程項目は開始日時順、開始日時が同じ場合は終了日時順で表示する
- Androidウィジェットに左右ボタンを追加し、キャッシュ済みの別旅行へ切り替えられるようにする
  - キャッシュ範囲より前後へ移動しようとした場合はDBから追加取得する
- Androidウィジェットに更新ボタンを追加し、更新ボタンを押した場合のみDBから最新データを取得する
  - 通常表示とキャッシュ範囲内の左右切り替えではDB通信せず、保存済みキャッシュを表示する
  - 更新成功時はキャッシュとウィジェット表示を更新する
  - 更新失敗時はキャッシュ済みの内容を維持し、最終更新日時またはエラー状態を表示する
- Androidウィジェット用のキャッシュ保存先と保存形式を決める
  - Flutter側とAndroidネイティブ側の両方から参照できる形式にする
  - ログアウト時や対象グループ変更時に古いキャッシュを削除する
  - 対象グループIDも端末ローカルの設定としてキャッシュと同じ管理範囲で扱う
