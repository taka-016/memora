# Coding Preparation

コード変更に入る前に、このリポジトリで求められる準備を揃える。

## 基本方針

- 質問への回答のみが目的で、実装やファイル変更を伴わない場合はこのスキルを適用しない。
- ユーザーが`todo`確認の省略を明示した場合や、指示の中に`todo`作成自体が含まれる場合は、その指示を優先する。

## 手順

1. ユーザーからの指示に該当する作業が`docs/todo.md`に存在するか確認する。
2. 該当する作業が存在しない場合は、作業を中断してユーザーに確認する。
3. `tree lib test`を実行して、アプリケーションコードとテストコードのディレクトリ構造を確認する。
4. `AGENTS.md`の「設計資料」に挙がっている資料を確認し、少なくとも`docs/app_spec.md`からアプリケーション仕様を把握する。

## 実行上の注意

- `todo`確認を省略しない限り、`docs/todo.md`未記載のまま実装に入らない。
- 差分が発生する作業を始める前に、`AGENTS.md`の基本ルールにあるブランチ作成条件を満たしていることを確認する。
