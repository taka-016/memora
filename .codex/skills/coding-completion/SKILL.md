# Coding Completion

コード変更を行うときは必ず最初からこのスキルを使う前提で進め、最後に検証、必要な`todo`更新、ステージング、コミット、プッシュ、PR作成まで一貫して進める。

## 基本方針

- `coding` や `coding-preparation` を使う作業では、完了手順として必ずこのスキルも併用する。
- ユーザーが`todo`更新の省略を明示した場合は、その指示を優先する。
- ユーザーがコミット、push、PR作成の省略を明示していない限り、完了条件にはそれらを含める。

## 手順

1. `./check.sh`を実行し、エラーが残っていないことを確認してから完了作業に進む。
2. ユーザーが`todo`更新の省略を指示していない場合、作業に対応する`docs/todo.md`の項目を`docs/done.md`に移動する。
3. ユーザーが`todo`更新の省略を指示していない場合、今回作業指示された`docs/todo.md`の項目が`docs/done.md`に存在し、`docs/todo.md`側に残っていないことを確認する。
4. 作業に関連するファイルを`git add <対象ファイル>`でステージングに追加する。`git add .`は使わない。
5. `AGENTS.md`の「コミット形式」に従ってコミットし、pushし、プルリクエストを作成する。
6. `todo`更新を行った場合、プルリクエストの説明には`docs/done.md`に移動した作業対応項目を記載する。

## 実行上の注意

- 検証が失敗している状態で完了扱いにしない。
- ステージングは必ず対象ファイルを明示して行う。
- コミット形式やPR説明は、`AGENTS.md`の定義を優先する。
