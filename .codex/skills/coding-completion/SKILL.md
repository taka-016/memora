# Coding Completion

コード変更の完了時は、検証、`todo`更新、ステージング、コミット、プッシュ、PR作成まで一貫して進める。

## 手順

1. `./check.sh`を実行し、エラーが残っていないことを確認してから完了作業に進む。
2. 作業に対応する`docs/todo.md`の項目を`docs/done.md`に移動する。
3. 今回作業指示された`docs/todo.md`の項目が`docs/done.md`に存在し、`docs/todo.md`側に残っていないことを確認する。
4. 作業に関連するファイルを`git add <対象ファイル>`でステージングに追加する。`git add .`は使わない。
5. `AGENTS.md`の「コミット形式」に従ってコミットし、pushし、プルリクエストを作成する。
6. プルリクエストの説明には、`docs/done.md`に移動した作業対応項目を記載する。

## 実行上の注意

- 検証が失敗している状態で完了扱いにしない。
- ステージングは必ず対象ファイルを明示して行う。
- コミット形式やPR説明は、`AGENTS.md`の定義を優先する。
