# Coding Completion

コード変更を完了する前に、このリポジトリで求められる締め手順を揃える。

## 基本方針

- ユーザーが`todo`更新の省略を明示した場合は、その指示を優先する。
- このスキルは、`coding` で必要なコード変更とコミットが完了していることを前提に使う。
- 検証や最終確認で修正が必要になった場合は完了作業を中断し、`coding` に戻って修正とコミットを完了してからやり直す。
- ユーザーがpush、PR作成の省略を明示していない限り、完了条件にはそれらを含める。

## 手順

1. `./check.sh`を実行し、エラーが残っていないことを確認してから完了作業に進む。エラーが出た場合は`coding`に戻る。
2. ユーザーが`todo`更新の省略を指示していない場合、作業に対応する`docs/todo.md`の項目を`docs/done.md`に移動する。
3. ユーザーが`todo`更新の省略を指示していない場合、今回作業指示された`docs/todo.md`の項目が`docs/done.md`に存在し、`docs/todo.md`側に残っていないことを確認する。
4. `git status`と直近のコミットを確認し、`coding`で必要なコミットが完了していることを確認する。未コミットの修正が必要なら`coding`に戻る。
5. pushし、プルリクエストを作成する。
6. `todo`更新を行った場合、プルリクエストの説明には`docs/done.md`に移動した作業対応項目を記載する。

## 実行上の注意

- 検証が失敗している状態で完了扱いにしない。
- `coding-completion`内で新たな修正差分を抱えたまま完了させない。差分調整が必要なら`coding`へ戻す。
- PR説明は、`AGENTS.md`の定義を優先する。
